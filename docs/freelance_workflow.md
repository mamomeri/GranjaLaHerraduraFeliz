# Flujo de Trabajo Freelance

## 1. Recepción del levantamiento de información

- Recibes el objetivo del cliente.
- Identificas procesos clave y problemas.
- Analizas los casos de uso.

## 2. Generación de documentación base

### `README.md`
- Resumen del sistema.
- Objetivos.
- Alcance.

### `requirements.md`
- Requisitos funcionales.
- Requisitos no funcionales.
- Suposiciones y restricciones.

### `architecture-overview.md`
- Descripción de las capas.
- Tecnologías involucradas.
- Relaciones entre componentes.

## 3. Elaboración del `TODO.md`

### Etapas principales
- **Etapa 0**: Preparación del proyecto.
- **Etapa 1**: MVP funcional.
- **Etapa 2**: Mejoras.
- **Etapa 3**: Funcionalidades avanzadas.

### Consideraciones para freelance
- Priorización de entregables.
- División del sistema por hitos.
- Identificación del MVP.
- Identificación de fases futuras.

## 4. Ajuste según contexto freelance

- Definición de plazos.
- Estimación de prioridades.
- Presentación de versiones parciales.
- Clarificación del alcance.

## 5. Ciclo continuo de trabajo

### Ciclo de refinamiento
- Implementas las etapas iniciales.
- Realizas pruebas parciales.
- Revisas el avance con el cliente.
- Ajustas el TODO y plan de trabajo.

### Comunicación con el cliente
- Informas del avance.
- Presentas nuevas ideas y mejoras.
- Evalúas cambios de alcance.

---

Este flujo te permite trabajar como un equipo organizado incluso siendo freelance, manteniendo una estructura clara para cada entrega y proyecto.
